# Documentation Overview

This directory contains design-time, architectural, and governance documentation
for the project. These documents describe intent, constraints, and risk boundaries,
and are not implementation instructions.

## Reading Order (Recommended)

1. WORKBENCH_OVERVIEW.md  
   High-level purpose and conceptual model of the system.

2. ARCHITECTURE.md  
   Structural layout of the application and component boundaries.

3. WORKFLOW.md  
   How components interact during normal operation.

4. SESSION_LIFECYCLE.md  
   Definition of session boundaries and state transitions.

5. THREAT_MODEL.md  
   Identified risks, assumptions, and mitigations.

6. TRACEABILITY_MODEL.md  
   Data flow visibility and limitation strategy.

7. COMPLIANCE&ABUSE_PREVENTION.md  
   Safeguards, constraints, and misuse prevention considerations.

8. LEGAL_POSITIONING.md  
   Non-operational legal framing and scope clarification.

9. WHITEPAPER.md  
   Extended rationale, philosophy, and long-form explanation.

## Scope

All documents in this directory are informational and descriptive.
They do not instruct or encourage misuse, surveillance, or policy violations.
